# 定时摄取和回顾

定时任务让知识摄取从“想起来才做”变成“稳定流入”。

## 推荐任务

| 任务         | 频率 | 输出                      |
| ------------ | ---- | ------------------------- |
| 抓取 RSS     | 每天 | `Inbox/` 摘要             |
| 整理手动输入 | 每天 | 待整理清单                |
| 更新主题 MOC | 每周 | MOC 修改建议              |
| 周回顾       | 每周 | `Notes/周回顾-YYYY-WW.md` |

## Hermes 任务模板

```text
任务：每日知识摄取
触发：每天 09:00
输入：RSS 列表和待处理 Inbox
处理：抓取、去重、摘要、状态标记
输出：Obsidian Inbox 和一条摘要通知
失败：写日志，不重复刷屏
```

## 验收标准

- 每次任务都有日志。
- 每条内容都有来源。
- 不确定内容进入 `review`。
- Bot 推送摘要不超过一屏。
- 每周有人类回顾。

完成这一章后，你就有了一条完整路径：本地知识库规则由 Claude Code 维护，长期摄取由 Hermes 触发，两个 Skill 分别负责摄入和整理。
