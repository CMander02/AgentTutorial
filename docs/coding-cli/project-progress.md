# 使用 /plan、/goal 和历史记录

长期任务不能只靠一轮 prompt。你需要让 Agent 明确目标、计划和当前状态。

## /plan

`/plan` 适合用在改动前：

```text
/plan
请阅读当前文档结构，提出一个面向新手的章节重构方案。先不要修改文件。
```

适合：

- 信息架构设计
- 大范围重构
- 多文件修改
- 需要先讨论取舍的任务

## /goal

`/goal` 适合用来声明本轮工作的最终目标：

```text
/goal 把当前文档改成面向新手的 Agent 教程，并通过 mkdocs strict 构建。
```

适合：

- 长任务
- 需要持续推进的修复
- 有明确完成标准的工作

## 历史记录和交接

新手要养成三个习惯：

- 修改前让 Agent 总结当前状态。
- 修改后让 Agent 写清楚改了什么。
- 中断或切换工具前，让 Agent 留下下一步。

## 常用提示词

```text
请先复盘当前目标、已完成事项、剩余风险，然后继续。
```

```text
请把当前工作整理成交接说明，包含文件、命令、验证结果和下一步。
```
