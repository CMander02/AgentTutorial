# 第一次交互和提问方式

和 Coding CLI 交互时，不要只说“帮我做一下”。新手最重要的是学会给边界、目标和验收标准。

## 一个好请求的结构

```text
目标：我要完成什么。
范围：可以读写哪些文件。
限制：不要做什么。
验收：怎样算完成。
节奏：先计划，确认后再修改。
```

## 示例

差的问法：

```text
帮我优化一下项目。
```

好的问法：

```text
请阅读 README、mkdocs.yml 和 docs/ 目录，判断这个教程的信息架构是否适合新手。
先给修改建议，不要改文件。
```

更适合执行的问法：

```text
按刚才的建议修改 docs/ 目录。只改文档，不改代码。完成后运行 uv run mkdocs build --strict。
```

## 常见交互模式

| 模式 | 适合任务 |
| --- | --- |
| 先读后答 | 理解项目、诊断问题 |
| 先计划后改 | 文档改版、代码重构 |
| 小步执行 | 风险较高的批量修改 |
| 执行后验证 | 构建、测试、检查链接 |
