# 正式开始之前

!!! abstract "导言"

    Coding CLI 会读取文件、运行命令和修改项目。
    因此新手在正式使用前，需要先新建一个干净文件夹，作为相对隔离的环境。
    本章节同时阐述一些默认观念。

## 目标 {#goal}

完成本章后，你应该拥有：

- 一个专门练习 Agent 的本地目录。
- 一个能打开的终端。
- 一个能记录改动的 Git 仓库。
- 一套遇到风险时先暂停的习惯。

## 准备练习目录 {#workspace}

建议把练习项目放在短路径下。Windows 用户可以使用类似路径：

```powershell
C:\Work\AgentPractice
```

路径短一些，能减少空格、中文路径、同步盘和权限带来的干扰。等你熟悉工作流后，再把经验迁移到真实项目。

## 学会看三类输出 {#outputs}

Coding CLI 工作时，经常会给出三类信息：

| 输出     | 你需要看什么             |
| -------- | ------------------------ |
| 文件列表 | 它是否读到了正确目录     |
| 命令输出 | 命令是否成功，错误在哪里 |
| Git diff | 它到底改了哪些内容       |

先学会看输出，再让 Agent 做更长的任务。

## 初始化 Git {#git}

如果练习目录还没有 Git 仓库，可以运行：

```powershell
git init
git status
```

每次让 Agent 修改前，先看当前状态：

```powershell
git status --short
```

如果已有未提交改动，需要先知道这些改动来自谁、是否需要保留。

## 建立安全边界 {#boundaries}

给 Agent 任务时，先说清楚边界：

```text
请只阅读当前目录下的 README 和 docs/，先总结项目结构。不要修改文件，不要删除文件，不要运行安装命令。
```

涉及删除、迁移、批量重命名、提交、推送、发消息和联网操作时，应要求它先给计划，并等待确认。

??? warning "高风险操作"

    删除目录、批量移动文件、覆盖配置、发送外部消息、修改账号权限和处理密钥，都应先停下来检查计划。

## 验证 {#check}

完成准备后，执行这三步：

```powershell
pwd
git status --short
Get-ChildItem
```

你应该能说出当前目录在哪里、Git 是否干净、目录里有哪些文件。

## 常见误区 {#pitfalls}

??? warning "直接在重要目录里练习"

    第一次练习应使用可回滚的样本项目。真实工作目录里通常有配置、密钥、同步文件和未提交改动。

??? warning "只看 Agent 的总结"

    Agent 说完成了任务后，还要看文件 diff、命令输出和页面效果。
