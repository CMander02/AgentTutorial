# 接入 QQ / 微信 / 飞书 Bot

Bot 接入的本质是：把外部聊天消息转成 Hermes 能处理的事件，再把处理结果发回聊天平台。

## 通用结构

```text
聊天平台 -> Bot 网关 -> Hermes -> Agent 工作流 -> 回复/入库/推送
```

## 接入前要确认

- 平台是否允许 Bot。
- token 和密钥如何保存。
- 消息格式是什么。
- 是否有频率限制。
- 群聊里如何避免循环回复。
- 哪些消息应该触发 Agent，哪些不触发。

## 三个平台的关注点

| 平台 | 关注点                           |
| ---- | -------------------------------- |
| QQ   | 接入方式、权限、群聊触发规则     |
| 微信 | 账号形态、风控、消息限制         |
| 飞书 | 应用权限、事件订阅、企业空间范围 |

## 新手建议

先做一个只回复固定文本的 Bot，再接入 Agent 工作流。这样可以把平台接入问题和 Agent 逻辑问题分开排查。
