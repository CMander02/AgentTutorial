# 龙虾是什么

本文里的“龙虾”是一个应用形态的简称：一个常驻式 Agent 入口，通常有聊天界面、工具后台、Bot 接入和定时任务。

你可以把它理解成：

```text
网页/聊天入口 + Agent 工作流 + 工具调用 + 长期运行服务
```

## 和 Coding CLI 的区别

| 维度     | Coding CLI           | 类龙虾应用                   |
| -------- | -------------------- | ---------------------------- |
| 入口     | 终端                 | 网页、聊天软件、Bot          |
| 运行位置 | 本地项目             | 服务器或常驻应用             |
| 适合任务 | 代码、文档、本地文件 | 定时任务、消息处理、长期摄取 |
| 用户交互 | 人主动发起           | 人主动或系统自动触发         |

## 为什么需要它

Coding CLI 很适合搭建和调试工作流，但它不适合一直开着等消息。类龙虾应用补上的是“常驻入口”和“长期运行”。
