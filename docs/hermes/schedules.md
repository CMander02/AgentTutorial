# 配置定时任务

定时任务是 Hermes 这类常驻应用的重要能力。它让 Agent 不需要等你手动触发，也能周期性工作。

## 常见场景

- 每天早上抓取新闻并总结。
- 每周整理知识库新增内容。
- 每小时检查 RSS 或网页更新。
- 每晚生成待办和回顾。

## 配置原则

- 任务要小，不要一次做太多事。
- 每个任务都要有日志。
- 失败要有退避和上限。
- 输出要有固定位置。
- 重要任务要能手动重跑。

## 任务模板

```text
名称：每日内容摄取
触发：每天 09:00
输入：RSS 列表 / 网页列表
处理：抓取 -> 去重 -> 总结 -> 分类
输出：Obsidian Inbox / 每日摘要
失败处理：记录日志并下次重试
```
