# 在云服务器部署 Hermes

这一页先给部署检查表。具体命令应以你的 Hermes 仓库 README 为准。

## 服务器准备

- 一台 Linux 云服务器。
- 一个普通用户，不建议长期用 root 运行服务。
- Git、运行时环境和进程管理工具。
- 可访问模型服务和消息平台 API。

## 配置准备

通常需要：

- 模型供应商配置
- Bot 平台 token
- 数据目录
- 日志目录
- 定时任务配置
- 反向代理或 HTTPS 配置

## 部署流程

```text
拉取代码 -> 安装依赖 -> 写配置 -> 本地启动测试 -> 配置守护进程 -> 配置日志 -> 配置备份
```

## 验证标准

- 服务能重启。
- 日志能看到。
- Bot 能收到测试消息。
- 定时任务能触发一次。
- 失败时不会无限重试刷屏。
