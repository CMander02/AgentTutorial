# 当前两类热门形态

现在最值得新手优先理解的 Agent 形态有两类：Coding CLI 和 类 OpenClaw Agent

## Coding CLI

Coding CLI 是运行在计算机终端里的 Agent。它可以进入一个项目目录，读文件、改文件、跑命令、提交变更。

适合：

- 写代码
- 改文档
- 维护本地知识库
- 跑数据脚本
- 做项目级长期任务

这类工具的关键不是“会写代码”，而是它能和本地文件系统、命令行、Git 历史一起工作。

## 类龙虾应用

本文里的“类龙虾应用”指一种常驻式 Agent 入口：它更像一个长期在线的助手，可以接入 QQ、微信、飞书等聊天入口，也可以在服务器上跑定时任务。

适合：

- 定时抓取内容
- 接收聊天消息并处理
- 推送提醒
- 把零散信息送入知识库
- 把 Agent 工作流服务化

## 两者怎么配合

最容易落地的组合是：

```text
Coding CLI 负责构建和维护工作流
Hermes / 类龙虾应用负责长期运行和消息入口
```

本教程后半部分会按这个组合做两个实战。
